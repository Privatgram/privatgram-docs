# Introduction

Welcome to **PRIVATGRAM®**, a secure enterprise messaging platform designed for organizations requiring confidential communications and centralized administration.

This guide explains how to begin using PRIVATGRAM, communicate securely, manage devices and follow recommended security practices.

---

# Intended Audience

This guide is intended for:

- Employees
- Managers
- Government personnel
- Banking personnel
- Enterprise users

No technical knowledge is required.

---

# First Login

Your organization administrator provides:

- Organization
- Username
- Temporary Password

Open the PRIVATGRAM application.

Enter:

- Organization
- Username
- Password

Select:

**Sign In**

The application will verify your credentials and register the current device.

---

# Device Registration

Each device must be individually registered.

During registration the application:

- Creates a local cryptographic identity
- Registers the device
- Initializes secure communication
- Stores cryptographic material locally

Each registered device becomes an independent trusted endpoint.

---

# User Profile

Users may configure:

- Display Name
- Profile Photo
- Language
- Notification Preferences

Certain profile fields may be managed centrally by the organization.

# Home Screen

The main interface includes:

- Chats
- Contacts
- Organizations
- Settings
- Notifications

# Contacts

Contacts may include:

- Individual users
- Departments
- Organizational groups

Availability depends on administrator configuration.

---

# Starting a Conversation

To begin a conversation:

1. Open Contacts
2. Select a user
3. Open chat
4. Type a message
5. Press Send

The application automatically establishes a secure communication session where required.

---

# Secure Messaging

Messages are protected using end-to-end encryption.

The application automatically performs:

- Session establishment
- Encryption
- Authentication
- Secure transmission
- Local storage

These operations are transparent to the user.

---

# File Sharing

Users may securely exchange files.

Typical supported content includes:

- Documents
- Images
- PDF files
- Office documents

Access to transferred files follows organizational security policies.

---

# Message Status

Messages may display delivery status.

Typical states include:

- Sending
- Delivered
- Read

Availability depends upon organizational configuration.

---

# Notifications

Users may configure:

- Sound
- Vibration
- Desktop notifications
- Mobile notifications

Notification behavior may be controlled by enterprise policy.

---

# Multiple Devices

A user may access PRIVATGRAM from multiple authorized devices.

Each device maintains its own secure cryptographic state.

Administrators may revoke individual devices without affecting other authorized devices.

---

# Device Management

Users may review:

- Registered devices
- Active sessions

If an unknown device appears:

1. Contact your administrator.
2. Revoke the device if permitted.
3. Change your password if instructed.

---

# Security Indicators

Users should pay attention to:

- Unknown devices
- Unexpected login notifications
- Repeated authentication requests
- Security warnings

Report suspicious activity immediately.

---

# Password Management

Users should:

- Use strong passwords
- Never share passwords
- Change temporary passwords
- Follow organizational password policies

Where supported, additional authentication methods may be available.

---

# Logging Out

To sign out:

1. Open Settings.
2. Select **Sign Out**.
3. Confirm the action.

Signing out removes the active session from the current device.

---

# Lost or Stolen Device

If a device is lost:

1. Inform your administrator immediately.
2. Request device revocation.
3. Change your password if instructed.

Rapid reporting reduces organizational risk.

---

# Security Best Practices

Recommended practices include:

- Lock your device when unattended.
- Install operating system updates.
- Never share authentication credentials.
- Verify recipient identity before sending sensitive information.
- Report suspicious behavior immediately.

---

# Privacy

PRIVATGRAM is designed to support confidential organizational communication.

Users should nevertheless comply with:

- Organizational policies
- Information classification rules
- Internal security requirements

End-to-end encryption does not replace responsible handling of sensitive information.

---

# Troubleshooting

### Cannot Sign In

Possible causes:

- Incorrect password
- Network connectivity
- Disabled account

Contact your administrator if the issue persists.

---

### Device Not Registered

Possible causes:

- Device awaiting approval
- Organization policy
- Network issues

Retry registration or contact your administrator.

---

### Messages Not Delivered

Check:

- Internet connectivity
- Recipient availability
- Organization status

If the problem continues, contact technical support.

# Frequently Asked Questions

### Can administrators read my encrypted messages?

Administrative functions are designed separately from message protection.

Refer to the Security documentation for details regarding the platform security architecture.

---

### Can I use multiple devices?

Yes.

Each authorized device maintains an independent secure session.

---

### What happens if I lose my phone?

Your administrator can revoke the affected device without impacting other registered devices.

---

### Can I transfer files?

Yes.

File transfer is subject to organizational security policies.
